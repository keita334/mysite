# FilamentMaker

`FilamentMaker` は、フィラメント作成に関するファイルを管理するディレクトリです。

## 目的

- フィラメント作成プロセスの記録
- 設定値や実験結果の整理
- 関連データや手順の再利用

## 想定構成

- `settings/`: 温度・速度・材料などの設定
- `logs/`: 試作ログ、結果メモ
- `assets/`: 参考画像、図、補助データ

## 使い方

1. 作業前に設定を記録する
2. 作業ごとに結果をログとして残す
3. 再現可能な条件をREADMEやログに追記する

## メモ

必要に応じて、実際の運用に合わせてこのREADMEを更新してください。

## 検証ログ（2026-04）

### 目的

- `ID-C0512NX` + `IT-007R` + PC で測定値をCSV取得する

### 実施内容

1. IT-007R仕様を確認
- 通信条件: `2400 bps / 8N1 / flow control none`
- PC要求文字候補: `"1"` または `"1\\r"`
- 取得間隔は `1秒以上` 推奨

2. ID-C0512NX側設定を確認
- `outPut` メニューで `d1 / d2` を切り替えて試験
- どちらでも受信は発生せず

3. PC側受信試験
- `capture_it007r.py` 実行で連続 `timeout`
- `miniterm` で受信待ちして `DATA` 押下しても無反応

4. RS-232電圧を実測
- `pin 2,6,8` は約 `-9V`
- `pin 4(DTR), 7(RTS)` は `0V`

### 結論

- IT-007Rは `pin 4(DTR)` と `pin 7(RTS)` からの給電前提
- 使用中の `CableCreation PL-2303` 構成では4/7に電圧が出ず、IT-007Rが駆動できていない可能性が高い

### 次アクション

1. `DTR/RTS` が有効なUSB-RS232変換器へ変更（例: FTDI系 / Keyspan系）
2. `DB9 全結線ストレート(9芯)` ケーブルを使用
3. 交換後に `pin 4,7` 電圧再測定 -> 受信再試験
