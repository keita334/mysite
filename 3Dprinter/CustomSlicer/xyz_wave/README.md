# STL XYZ Wave Slicer (Blender)

`xywave/custom_slicer_stl_xywave_addon.py` の構成をベースにした `xyz_wave` 版です。

## ベース方針
- 断面抽出・ループ整列・法線算出: XY Wave実装を踏襲
- UI/オペレーター構成: XY Wave実装の流れを踏襲
- Z方向連続性: 層高さに応じて接線方向へ位相を自動シフト
- 接線傾き: 変位方向を `法線 + 接線` の合成方向へ傾ける（UI項目は廃止し内部固定）

## 使い方
1. Blenderで `xyz_wave_blender.py` を開いて実行
2. `View3D > Sidebar > Custom Slicer > STL XYZ Wave Slicer`
3. `対象メッシュ` にSTLオブジェクトを指定
4. `XYZ Waveプレビュー更新` でパス確認
5. `XYZ Wave G-code保存` で出力

## 備考
`zwave` フォルダには `.blend` はありますが、Pythonソースは確認できなかったため、
コードベースは `xywave` 側を主に流用しています。

## 開発ログ（これまでの流れ）
このプロジェクトでは、当初の「輪郭沿いのsin波生成」から、層間接触の安定化を目的に段階的に実装を更新しました。

1. 初期実装
- STL輪郭抽出、ループ再サンプリング、位相整列、プレビュー/G-code出力を実装。
- 波は輪郭接線・表面法線から導出した方向に沿って付与。

2. 接触要件の強化
- 「前層基準で接触させる」要件に合わせ、層間の位相アンカーを導入。
- 1層目フラット時の接触条件、開始点（山開始）の固定などを調整。

3. 浅い傾斜での浮き対策
- 層ピッチを傾斜依存で調整。
- 前層参照をパラメトリック再評価中心から、実生成ポイント参照（`final_points`）へ切替えて累積誤差を低減。

4. 急傾斜でのめり込み対策
- ガードを強化し、近傍候補に対して保守的に交差を回避。
- 補正上限を傾斜依存で制御。

5. 補正方式の見直し
- 層全体一括シフトだけでは両立が難しいため、局所補正＋平滑化へ移行。
- 「接触させたい側（山/谷）」を切替え可能なロジックへ段階的に整理。

6. 最新方針（現状）
- 山接触モードに合わせて、位相アンカー・接触スコア・残差補正を山基準へ反転。
- `contact_overlap=0` 時は「フィラメント同士が接する」ことを優先。
- 依然として形状依存のチューニング余地があるため、係数はモデル別微調整が前提。

## 現在の注意点
- 接触はモデルの局所傾斜・曲率・周波数設定に強く依存します。
- `path_steps` が低いと局所補正が荒くなり、接触判定が不安定になります。
- `z_amplitude` と `z_frequency` を大きくすると接触条件が急激に厳しくなります。
